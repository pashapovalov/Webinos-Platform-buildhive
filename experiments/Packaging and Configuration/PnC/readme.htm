Introduction

The PnC library is an implementation of the W3C packaging and configuration specification (http://www.w3.org/TR/widgets/).

It forms part of the BONDI reference implementation and is built using standard C++ in conjunction with the libxml2 xml parser.

Functionality includes:

  • Extracting a widget from an archive
  • Parsing and processing the widget configuration document
  • Support for multiple locales
  • Project file currently supports win32 and WM 6 builds

Coming soon - digital signature support.

Although it is intended to be a cross-platform library, there are a couple of things remaining that need to be sorted out, namely:

  • cross-platform support for converting to/from multi-byte character sets
  • a cross-platform makefile

Building the library

Extract the ZIP file to your local disk, including the libxml2 folder.

There is currently only a Visual Studio 2005 project file. Open the PnC.vcproj file in Visual Studio and build the library.

Using the library

After building the library you can use it in your project by making the PnC.h file available on your include paths and statically linking with the PnC.lib.

Here is an example usage:



	#include "PnC.h"

	CPackageLocales locales(L"en-gb");

	CPnC package = new CPnC(locales);
	
	if (package->LoadZip(L"C:\\MyWidgets\\coolWidget.wgt"))
	{
		// Widget loaded successfully - you can use the package.
		std::wstring startFilePath;
		if (package->GetStartFile(startFilePath))
		{
			
		}
	}
	else
	{
		// Widget failed to load - find out why.
		std::vector< std::wstring > faults;
		package->GetFaults(faults);
		
		// Display or log the faults.
	}
